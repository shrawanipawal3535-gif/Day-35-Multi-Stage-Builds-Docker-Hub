# Day-35-Multi-Stage-Builds-Docker-Hub


* Multi-stage builds are used to create **small and efficient Docker images**
* It divides the process into **build stage and final stage**
* Only the required files are copied to the final image
* This helps remove unnecessary tools and reduces image size
* Docker Hub is a cloud platform to **store and share Docker images**
* Images are pushed using **tags like v1, latest for version control**
* We can pull images anytime on any system using `docker pull`
